# Dashboards und Kurzberichte

Ein Dashboard ist eine Sammlung von Berichten in Miniaturansicht, die Reportlets genannt werden. Ein Dashboard ist besonders nützlich, wenn es verwandte Reportlets enthält, die Ihnen einen vollständigen Überblick über bestimmte Aspekte Ihrer Site geben, wie Suchmethoden, Besucher-Profil usw.

## Dashboards und Kurzberichte {#concept_8CD3ACA2830A4994A68A31D8773B57E0}

Das Dashboard ist eine Sammlung von Berichten in Miniaturansicht, die *`reportlets`*. Ein Dashboard ist besonders nützlich, wenn es verwandte Reportlets enthält, die Ihnen einen vollständigen Überblick über bestimmte Aspekte Ihrer Site geben, wie Suchmethoden, Besucher-Profil usw.

Sie können einem Dashboard die meisten Marketing-Berichte hinzufügen, einschließlich grafikintensiver Berichte wie [!UICONTROL Fallout Report], [!UICONTROL Conversion Funnel Report]und [!UICONTROL Pathfinder Report].

Sie können auch ein Dashboard als Ihre Landingpage festlegen, Dashboard für andere Benutzer freigeben und sie für den Versand planen. Wenn Sie kein Dashboard (oder Lesezeichen) als Landingpage festlegen, wird das [!UICONTROL My Recommended Reports] Dashboard angezeigt. **[!UICONTROL My Recommended Reports]** zeigt den **[!UICONTROL Key Metrics]** Bericht sowie Ihre fünf am häufigsten angezeigten Berichte an. Es handelt sich um eine dynamische Übersicht der tatsächlich am häufigsten angezeigten Berichte.

Beachten Sie, dass einige häufig angezeigte Berichte nicht mit Dashboards versehen werden können und nicht angezeigt werden. Dazu gehören:

* Berichte zur Anomalieerkennung
* Berichte zur Analyse von Beiträgen
* Trichteranalysen
* Pathfinder-Berichte
* Echtzeitberichte
* Andere Dashboards

>[!NOTE] Das **[!UICONTROL Site Overview]** Dashboard wird nicht mehr in Reports &amp; Analysen aufgeführt. Es gibt jedoch noch einige Fälle, in denen einige oder alle zugehörigen Reportlets angezeigt werden.

* Wenn Sie beispielsweise nur drei häufig angezeigte Berichte haben, nimmt Reports &amp; Analysen zwei Berichte aus dem Dashboard &quot;Site-Übersicht&quot;in Anspruch, um das **[!UICONTROL My Recommended Reports]** Dashboard abzuschließen.
* Neue Report Suites mit Branding-Funktion werden zunächst auch die Reportlets &quot;Site-Übersicht&quot;enthalten, bis sie durch häufig angezeigte Berichte ersetzt werden. Trotzdem wird das Dashboard jetzt aufgerufen **[!UICONTROL My Recommended Reports]**.

Neben den Dashboards, die Sie erstellen, stehen jedem Benutzer die folgenden, voreingestellten Dashboards zur Verfügung:

**[!UICONTROL Components]>[!UICONTROL Dashboards]>[!UICONTROL Shared Dashboards]>[!UICONTROL Local Sites]**

Mit diesem anpassbaren Dashboard können Sie Reportlets in der bereitgestellten Vorlage ablegen.

**[!UICONTROL Components]>[!UICONTROL Dashboards]>[!UICONTROL Shared Dashboards]>[!UICONTROL Site Operations Dashboard]**

In diesem Dashboard erhalten Sie einen Überblick über die Schlüsselmetriken im Zusammenhang mit dem Betrieb Ihrer Website. Zu den Berichten zu diesem Dashboard gehören:

* Exitpages
* Bevorzugte Seiten
* Bevorzugte Site-Abschnitte
* KPI-/Kennzahlen-Reportlet
* Text-Reportlet
* Firmenzusammenfassungs-Reportlet

Verwenden Sie die [!UICONTROL Dashboard Manager] , um Dashboard zu bearbeiten und zu verwalten und sie für DirectAccess zu aktivieren.

Siehe [Verwalten von Dashboards](/help/analyze/reports-analytics/dashboard-manage.md).

## Dashboard erstellen {#task_54EFBED59BDC4418A919E6EF84AB9CFF}

In diesen Schritten wird beschrieben, wie Sie ein Dashboard erstellen.

<!-- 

t_dashboard_add.xml

 -->

Bevor Sie einem Dashboard einen Bericht (als Reportlet) hinzufügen, definieren Sie das Layout des Dashboards.

1. Öffnen von **[!UICONTROL Analytics]** > **[!UICONTROL Components]** > **[!UICONTROL Manage Dashboards]**.
1. Klicken Sie auf **[!UICONTROL Add Dashboard]**.
1. Geben Sie einen Namen für das Dashboard ein.
1. Klicken Sie auf **[!UICONTROL 3 x 2]** oder **[!UICONTROL 2 x 2]**, um festzulegen, wie viele Reportlets Sie auf der Dashboard-Seite anzeigen möchten.
1. Konfigurieren der Dashboard-Seite:

   * **[!UICONTROL Add Page]**: Fügt dem Dashboard eine leere Seite hinzu, auf die Sie Inhalte ziehen können, um Reportlets zu erstellen.
   * **[!UICONTROL Paper]**: Hier können Sie ein Papierformat angeben, z. B. Querformat, Hochformat und A4.
   * **[!UICONTROL Find Content]**: Ermöglicht die Suche nach Inhalten in den [!UICONTROL Add Content] Menüs und [!UICONTROL Dashboard Contents] Menüs.

1. Fügen Sie dem Dashboard verfügbare Inhalte hinzu, indem Sie Elemente in die Reportlet-Arbeitsfläche ziehen.

   Siehe [Erstellen eines Reportlets](/help/analyze/reports-analytics/dashboard.md#task_EC3AFBBAA51C45CEBAF632F841C305B3) und [Bearbeiten der Dashboard-Einstellungen](/help/analyze/reports-analytics/dashboard.md#task_90D7FAC1CC3E4DB786B4C87CC33B5459).
1. Klicken Sie auf **[!UICONTROL Save.]**

   Durch das Speichern eines Dashboards wird dieses im Menü **[!UICONTROL Dashboard]** verfügbar. The new dashboard is also available in the [!UICONTROL Dashboard Manager] ( **[!UICONTROL Favorites]** > **[!UICONTROL Dashboards]** > **[!UICONTROL Manager]**), where you can edit, organize, share, schedule, archive dashboards, and more. (Siehe [Verwalten von Dashboards](/help/analyze/reports-analytics/dashboard-manage.md).)

1. (Optional) Um das Dashboard als Landingpage festzulegen, klicken Sie auf **[!UICONTROL More Options]** > **[!UICONTROL Set as Landing Page]**.

## Reportlet erstellen {#task_EC3AFBBAA51C45CEBAF632F841C305B3}

In diesen Schritten wird beschrieben, wie Sie ein Reportlet erstellen. Nachdem Sie ein Reportlet erstellt haben, ist es für die Anzeige in einem Dashboard verfügbar.

<!-- 

t_dashboard_add_report.xml

 -->

1. Einen Bericht ausführen.
1. Klicken Sie auf **[!UICONTROL Dashboard.]**
1. Benennen Sie auf der [!UICONTROL Add Reportlet] Seite den Bericht und wählen Sie dann ein Dashboard aus **[!UICONTROL Place in Dashboard]**.
1. (Optional) Konfigurieren Sie den Datumsbereich.

   * **[!UICONTROL Rolling]**: Ändert das Datum im Zeitverlauf entsprechend der Zeitspanne (täglich, monatlich usw.). Wenn heute beispielsweise der 17. Januar ist, können Sie die Daten für den 15. bis 16. Januar festlegen. Wenn Sie dann am 27. Januar **[!UICONTROL Rolling]** die Option auswählen, zeigt das Reportlet Daten für den 25. bis 26. Januar an.
   * **[!UICONTROL Fixed]**: Verhindert, dass das Datum im Laufe der Zeit vorwärts verschoben wird.

1. (Optional) Verteilungsliste für Veröffentlichung außer Kraft setzen.

   **[!UICONTROL Publishing List Override]**: Wenn Sie diese Option aktivieren, wird die Report Suite, auf die in diesem Reportlet verwiesen wird, immer verwendet, wenn sie in einer Veröffentlichungs-Liste verteilt wird. Wenn Sie diese Option deaktivieren, ersetzt die in der Veröffentlichungs-Liste identifizierte Report Suite die Report Suite in diesem Reportlet.

1. Klicken Sie auf **[!UICONTROL Create New]**.

   The reportlet is added to the **[!UICONTROL Dashboard Contents]** menu in the dashboard editor.

## Inhalt zu einem Dashboard hinzufügen {#task_90D7FAC1CC3E4DB786B4C87CC33B5459}

In diesen Schritten wird beschrieben, wie Sie Inhalte aus anderen Dashboards und freigegebenen Dashboards hinzufügen. Sie können auch Inhalte aus benutzerdefinierten und externen Quellen hinzufügen, z. B. Text und Bilder.

<!-- 

t_dashboard_content.xml

 -->

1. Öffnen Sie ein Dashboard und klicken Sie auf **[!UICONTROL Layout]**. 
1. Click **[!UICONTROL Add Content]**, then drag items to the dashboard.

   The [!UICONTROL Add Content] menu displays reportlet content from other dashboards, legacy dashboards, and shared dashboards.

   >[!NOTE]
   >
   >Aktuell liegt die Obergrenze der Seitenzahlen in einem Dashboard bei 30.

   **Benutzerspezifische Reportlets**

   *Dateninhalt:*

   * Firmenzusammenfassung: Zeigt Seitenansichten aus mehreren, von Ihnen gewählten Report Suites und Metriken an.
   * Metrikmessung: Zeigt eine Messung an, die angibt, wo sich die Zahlen der Metrik im Vergleich zu den festgelegten Schwellenwerten befinden.

      Sie können eine Metrik, einen Diagrammtyp, einen Farbbereich sowie Schwellenwerte auswählen. Falls der Wert der Metrik den Schwellenwert größer als übersteigt, so wird dies durch die Messung in dem Reportlet angezeigt. Hierbei wird die Farbe über dem Feld größer als verwendet. Falls der Wert der Metrik unter dem Schwellenwert kleiner als liegt, so wird dies durch die Messung in dem Reportlet angezeigt. Hierbei wird die Farbe über dem Feld kleiner als verwendet. Die Werte, die Sie in diesen Feldern angeben, sind der zählbare Wert der Metrik, z. B. Anzahl der Ansichten, Dollarbeträge, Ansichten des Einkaufswagens usw. (Verwenden Sie keine Sonderzeichen.)
   * Report Suite Zusammenfassung: Zeigt die gewählte Metrik und deren Gesamtwert bzw. den höchsten oder den niedrigsten Wert für die Report Suite an.
   * Gebrauchszusammenfassung: Zeigt Daten über den Zugriff auf die Benutzeroberfläche durch Mitarbeiter in Ihrem Unternehmen an. In diesem Reportlet können Sie die Daten nach Benutzernamenzugriff, Berichtszugriff oder Report Suite-Zugriff anzeigen.
Durch Angabe von URLs können folgende Benutzerinhalt-Reportlets erstellt werden: Wenn eine Bild-URL oder eine andere Ressourcen-URL nicht mit https:// beginnt, sehen Internet Explorer-Benutzer möglicherweise eine Warnung zur Inhaltsmischung. Sie können die Warnung zur Inhaltsmischung in den Sicherheitseinstellungen Ihres Browsers deaktivieren.
   *Benutzerinhalt:*

   * Externer Bericht: Ermöglicht den Zusatz eines externen Berichts im .xml- und .csv-Format.
   * HTML: Ermöglicht den Zusatz eines benutzerspezifischen HTML-Reportlets. Die URL muss mit HTTP oder HTTPS beginnen. Andernfalls wird der Fehler `Specified URL could not be retrieved` angezeigt. Im Inhalt des Dokuments werden alle Tags mit Attributen, die die data:- und javascript:-Protokolle verwenden, entfernt. Skripts, Frames, Applets, Ereignishandler, Flash und andere integrierte Objekte werden entfernt. Wenn Ressourcen mithilfe eines Nicht-HTTPS-Protokolls angegeben werden, erhalten IE-Benutzer eine Warnung zur Inhaltsmischung.
   * Bild: Ermöglicht die Erstellung eines Dashboards aus einer Bild-URL. Wenn die URL das HTTP-Protokoll verwendet, zeigt Internet Explorer eine Warnung zur Inhaltsmischung an. Durch Verwendung einer URL mit HTTPS wird die Warnung entfernt. Bei allen anderen Protokollen wird der Fehler `Specified URL could not be retrieved` ausgegeben.
   * RSS: Ermöglicht den Zusatz eines RSS-Web-Feeds. Muss HTTP oder HTTPS entsprechen. Andernfalls wird der Fehler `Specified URL could not be retrieved` angezeigt.
   * Text: Sie können XHTML-Code zur Erstellung Ihrer eigenen Daten verwenden. Verwenden Sie für URLs HTTP oder HTTPS. Im Inhalt des Text-Reportlets verwendete Bilder mit HTTP-Protokoll bewirken, dass IE-Benutzer eine Warnung zur Inhaltsmischung erhalten. Bilder, die mit anderen Protokollen enthalten sind, werden nicht im Reportlet angezeigt.
   **Meine Dashboards**

   Liste der aktualisierten Dashboard, aus denen Sie Inhalte in das neue Dashboard verschieben können.

   **Legacy-Dashboards**

   Listet Ihre freigegebenen Dashboards auf, aus denen Sie Inhalte in das neue Dashboard verschieben können.

   **Freigegebene Dashboards**

   Listen veraltete Dashboard, aus denen Sie Inhalte in das neue Dashboard verschieben können. Ältere Dashboard sind hilfreich, wenn Sie die Formatierung von Dashboards gegenüber früheren Versionen beibehalten möchten.

   **Dashboard-Inhalt**

   Zeigt Elemente an, die Sie dem Dashboard bereits hinzugefügt haben.

1. Klicken Sie auf **[!UICONTROL Save.]**

## Dashboard- und Reportlet-Daten bearbeiten {#task_B460CCD70D9F40FCAC6BBC1C044CC460}

Sie können Dateneinstellungen auf Dashboard- oder Reportlet-Ebene ändern. Sie können beispielsweise die Report Suite ändern, die Report Suite sperren, Daten ändern, Segmente anwenden usw. Sie können ein Dashboard auch personalisieren, indem Sie die Vertraulichkeitserklärung Ihrer Firma einfügen und HTML-, XML-, Web API- und CSV-Daten in benutzerdefinierte Reportlets aufnehmen.

<!-- 

t_dashboard_edit.xml

 -->

**So bearbeiten Sie Dashboard- und Reportlet-Daten**

1. Click **[!UICONTROL Components]** > **[!UICONTROL Dashboards]** > *dashboard name* to open a dashboard.
1. Klicken Sie auf **[!UICONTROL Layout]**.

| Hierzu | Führen Sie dies aus |
|--- |--- |
| Report Suite eines Dashboards ändern | Klicken Sie auf das Menü in der Experience Cloud-Kopfzeile und wählen Sie dann eine Report Suite aus. |
| Ändern der Report Suite eines Reportlets | Klicken Sie im Reportlet auf den Namen der und wählen Sie anschließend aus dem Menü [!UICONTROL Report Suite]Report Suite eine Report Suite aus |
| Anwenden eines Segments auf ein Dashboard | Klicken Sie in der Experience Cloud-Kopfzeile auf [!UICONTROL Show Segments]und wählen Sie dann ein Segment aus. |
| Anwenden eines Segments auf ein Reportlet | Klicken Sie im Dashboard auf Layout, um ein Dashboard zu bearbeiten.   Klicken Sie im Reportlet auf den Report Suite-Namen, wählen Sie dann einen Wert aus dem Feld Segment und klicken Sie auf Aktualisieren. |
| Sperren einer Report Suite (verhindert, dass die Report Suite in einem Reportlet geändert wird) | Klicken Sie im Reportlet auf den Namen der Report Suite und aktivieren Sie dann [!UICONTROL Lock Report Suite]. Klicken Sie auf Aktualisieren.  |
| Ändern des Berichte | Klicken Sie für ein Dashboard auf den Kalender. (Alle Reportlets im Dashboard spiegeln die Änderung wider.)<br>Für ein Reportlet klicken Sie auf den Datums-Link, und konfigurieren Sie anschließend den Kalender. |
| Benennen eines Dashboards | Open a dashboard, then click  [!UICONTROL More] >  [!UICONTROL Rename]. |
| Anzeigen eines Dashboard-Archivs | Klicken Sie auf  [!UICONTROL More] >  [!UICONTROL View Archive]. |
| Festlegen des Dashboards als Landingpage | In a dashboard, click  [!UICONTROL More] > [!UICONTROL Set As Landing Page]. |
| Herunterladen eines Dashboards | In a dashboard, click  [!UICONTROL More] >  Download. |
| Drucken eines Dashboards | In a dashboard, click  [!UICONTROL More] >  Print. |
| Speichern eines Dashboards | Klicken Sie in einem Dashboard auf Speichern unter, und geben Sie einen Namen an. |

## Co-Branding eines Dashboards {#task_603BDE7700B945699AF5514C2DEB81F7}

In diesen Schritten wird beschrieben, wie Sie ein Bild zur Anzeige in einem Dashboard hochladen.

<!-- 

t_dashboard_branding.xml

 -->

1. **[!UICONTROL Analytics]** > **[!UICONTROL Admin]** > **[!UICONTROL Company Settings]**.
1. Klicken Sie auf der [!UICONTROL Company Settings] Seite auf **[!UICONTROL Co-Brand the Adobe Experience Cloud]**.
1. Klicken Sie auf **[!UICONTROL Enable Co-Branding]**.
1. Browse to upload the image, then click **[!UICONTROL Save.]**

   Die besten Ergebnisse beim Anzeigen des Bildes in einem Browser erzielen Sie, wenn Sie ein Bild mit 100 x 30 Pixel hochladen. Um optimale Ergebnisse bei der PDF-Ausgabe zu erzielen, laden Sie ein Bild der Größe 417 x 125 Pixel (300 dpi) hoch. Übergroße Bilder werden auf die Größe reduziert, wobei das Seitenverhältnis erhalten bleibt.

## Segmente mit Dashboards verwenden {#concept_ED030C3713D54D03971FB7B209285750}

Dashboard können, wie die meisten Berichte in Adobe Analytics, Segmente zum Abrufen der gewünschten Daten verwenden.

<!-- 

segments_dashboards.xml

 -->

Segmente können auf zwei Ebenen angewendet werden: auf ein ganzes Dashboard oder auf ein bestimmtes Reportlet.

* **Reportlet-Ebene**: Klicken Sie auf **[!UICONTROL Layout]** und dann auf die Report Suite des Reportlets, die Sie segmentieren möchten. Es wird ein modales Fenster angezeigt, in dem Sie Segmente hinzufügen oder ändern können, die vom Reportlet verwendet werden.
* **Dashboard-Ebene**: Klicken Sie im linken Navigationsbereich auf das Segment-Symbol, markieren Sie die gewünschten Segmente und klicken Sie auf Übernehmen. Die ausgewählten Segmente überschreiben und ersetzen alle Segmente auf Reportlet-Ebene.
