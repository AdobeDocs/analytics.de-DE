# Erste Schritte mit Ad Hoc Analysis {#concept_48936BA28FAE42DB81F1B2CD4726EB17}

>[!Important]
>Am 6. August 2018 kündigte Adobe die Absicht an, Ad Hoc Analysis einzustellen. Das Datum für das Ende des Produktlebenszyklus wird bekannt gegeben, sobald es verfügbar ist. Weitere Informationen dazu, welche Versionen von Java während dieses Zeitraums kompatibel sind, finden Sie unter [https://adobe.ly/discoverworkspace](https://adobe.ly/discoverworkspace).

Sie können sofortige, erweiterte Analysen auf der Website-Aktivität durchführen. Sie können mehrere Berichte gleichzeitig Ansicht leisten und Segmente über mehrere Dimensionen hinweg anwenden. Sie können die Daten sowohl aus der Mikro- als auch aus der Makroperspektive analysieren, um ihre Auswirkungen auf Ihre wichtigen Geschäftsmetriken Ansicht.

Mit diesen Funktionen können Sie Fragen zum Site-Traffic, zur demografischen Darstellung der Besucher, zum Umsatz und zur Produktbewegung beantworten. Anschließend können Sie Ihre Daten filtern, sortieren und segmentieren, um Antworten auf präzise Fragen zu erhalten. Die Ergebnisse werden fast sofort zurückgegeben, sodass Sie schnell die Auswirkungen einer Kombination von Faktoren analysieren können.

## Systemempfehlungen {#concept_6691331B45174290BD9B839806A9B52D}

Obwohl Berichte mit den meisten gängigen Webbrowsern angemessen funktionieren sollten, lassen sich die besten Ergebnisse bezüglich Gestaltung und Funktionsweise der Berichte auf Systemen erzielen, die gewisse Voraussetzungen erfüllen.

<!-- 

c_sys_reqs.xml

 -->

>[!NOTE] Ab Juli 2018 unterstützt Ad Hoc Analysis nur noch Java 8 oder höher. Wenn Sie Ad Hoc Analysis nach dem Release im Juli 2018 unter Java 7 ausführen, unterstützt Adobe die Implementierung von Ad Hoc Analysis nicht länger.

* Grafikkarte, die OpenGL 2.0 unterstützt
* Cookies: Erforderlich
* Betriebssystem: Windows und Mac OS.
* Macromedia Flash Player: Version 6 oder höher
* Bildschirmauflösung: 800 x 600 (1024 x 768 empfohlen)
* Farbtiefe: 16-Bit oder höher
* JavaScript: Aktiviert
* Java-Version: Java 1.7 oder höher (siehe Hinweis oben)

   Wenn Sie nicht die richtige Version von Java installiert haben, wird diese für Sie installiert. Wenn Sie eine nicht kompatible Version von Java installiert haben, lädt Ad-hoc-Analysen die Updates herunter und fordert Sie zur Installation auf.

## Anweisungen zum Java-Upgrade  {#section_E4C0C6492FF24636A0FF71A59331111D}

Ab Juli 2018 unterstützt Ad Hoc Analysis nur noch Java 8 oder höher. Wenn Sie Ad Hoc Analysis nach dem Release im Juli 2018 unter Java 7 ausführen, unterstützt Adobe die Implementierung von Ad Hoc Analysis nicht länger.

Unsere JAR-Dateien sind mit einer sicheren 256-Bit-Verschlüsselung signiert, die von älteren Java-Versionen als 1.7.0_76 nicht unterstützt wird. Dieses 256-Bit-Zertifikat verbessert die Sicherheit unserer Dienste für Sie.

Wenn auf Ihrem Rechner immer noch Java 7 installiert ist, müssen Sie vor dem Maintenance Release von Juli 2018 ein Upgrade durchführen. So geht’s:

* Wenn Sie Programme auf Ihrem Computer installieren dürfen:

   1. Gehen Sie zu https://www.java.com.
   1. Klicken Sie auf **[!UICONTROL Free Java Download]**.
   1. Klicken Sie auf **[!UICONTROL Agree and Start Free Download]**.
   1. Installieren Sie die neueste für Ihr Betriebssystem spezifische Java-Version.

* Wenn Sie **nicht** berechtigt sind, Programm auf Ihrem Computer zu installieren:

   1. Wenden Sie sich an Ihre IT-Abteilung, um die neueste Version von Java zu installieren.

## Über Experience Cloud anmelden {#task_128ED319F3AE49ED886EA3DFA8D0987F}

Schritte, die das Anmelden über [!DNL Experience Cloud] beschreiben.

<!-- 

t_login_suite.xml

 -->

1. Navigieren Sie in einem Browser zu [!DNL marketing.adobe.com].
1. Type your company name, your username, and your password. Then click **[!UICONTROL Sign In]**.
1. Klicken Sie auf **[!UICONTROL Adobe Analytics]** > **[!UICONTROL Tools]** > **[!UICONTROL Ad Hoc Analysis]**.

   Wenn Sie auf dieser Seite die Schaltfläche **Ad Hoc Analysis starten** nicht sehen, stellen Sie sicher, dass Ihr Administrator Sie in den Admin Tools der Gruppe *Ad Hoc Analysis-Lizenzanwender* hinzugefügt hat.
1. Klicken Sie auf **[!UICONTROL Launch Ad Hoc Analysis]**.
1. Speichern Sie die [!DNL discover.jnlp] Datei lokal.

   Sie können diese gespeicherte Datei ausführen, wann immer Sie Ad Hoc Analysis starten möchten.

## Projekte und Arbeitsbereiche {#concept_FAE346335B0347A192C6C806C775D72B}

Ein Projekt definiert den Datensatz, der importiert wird, einschließlich einer Report Suite und eines Datumsbereichs. Ein Projekt besteht aus einer beliebigen Anzahl von Berichten mit allen zugehörigen Metriken, Einstellungen, Dimensionen und Segmenten. Sie können ein neues Projekt starten, ein gespeichertes Projekt laden oder das automatisch gespeicherte Projekt laden.

<!-- 

c_projects.xml

 -->

Sie gruppieren Berichte in einer *Arbeitsfläche*. Ein Projekt kann mehrere Arbeitsbereiche und eine Arbeitsfläche mehrere Berichte enthalten. Die Beziehung zwischen diesen Elementen ist am besten als verschachtelt zu verstehen:

![](assets/project_workspace.png)

Sie können jeweils nur ein Projekt öffnen. Sie können jedoch mehrere Arbeitsbereiche in einem Projekt öffnen. In jedem Arbeitsbereich können mehrere Berichte geöffnet werden.

Der Standarddatumsbereich für ein neues Projekt ist [!UICONTROL Last 90 Days].

## Berichtsvorlagen {#concept_370F674C5B4C45368731AA801C5A45F8}

Vorlagen sind Ausgangspunkte für die Art der Analyse, die Sie durchführen möchten. Eine Vorlage kann eine leere Arbeitsfläche sein, z. B. ein Rang- oder Trichteranalysebericht. Oder die Vorlage ist ein Bericht, der Beginn mit Standardmetriken und -dimensionen enthält.

<!-- 

c_templates.xml

 -->

You can access templates when creating a project ( **[!UICONTROL File]** > **[!UICONTROL New Project]**), or by adding a workspace or report.

| Vorlage | Beschreibung |
|--- |--- |
| Rangansicht | Stellt eine leere Arbeitsfläche bereit, auf der Sie eine Tabelle erstellen können. So ordnet beispielsweise der Bericht „Seiten“ die Seiten Ihrer Website auf Grundlage des Datenverkehrs an, und in der Detailtabelle werden Prozentsätze und Zahlen für Metriken wie Seitenansichten und Umsatz angezeigt. |
| Trendansicht | Ermöglicht Ihnen, zu untersuchen, wie der Trend zu Konversionen und Ereignissen über eine bestimmte Zeitgranularität (Stunde, Tag, Woche, Monat, Quartal oder Jahr) während eines Berichte verläuft. |
| Gesamt | Ein Bericht auf Führungsebene, der Ergebniszahlen anzeigt. Es enthält Daten zu Gesamtumsatz, Seitenaufteilungen und Bestellungen. |
| Fallout | Ermöglicht den Aufbau eines Trichters, der Konversions- und Trichteranalyseraten zwischen Checkpoints anzeigt. Sie können beispielsweise die Trichteranalysepunkte eines Besuchers während eines Kaufprozesses verfolgen. |
| Fluss | Zeigt die häufigsten Pfade an, die Benutzer auf Seiten, Sitebereichen und Servern verwenden. |
| Konversionstrichter | Zeigt Konversionsprozentsätze zwischen bestimmten Metrik-Ereignissen an. Sie können diesen Bericht verwenden, um die Anzahl der Durchklicks, die zu Verkäufen führen, und die Anzahl der verkauften Einheiten zu verstehen. |
| Site-Analyse | Ein dreidimensionales Seitenpfadwerkzeug, das anzeigt, wie sich Besucher durch bestimmte Seiten und Ereignis bewegen. |
| Virtuelle Fokusgruppe | Besucht zufällig Ihre Besuche und zeigt Ihnen umfangreiche Daten zum Besuch. Sie können den Bericht verwenden, um Fokusgruppen aus Ihrer Besucher-Population zu erstellen. |

## Einen Bericht öffnen {#task_0AC455CDA198497AA546622FB05F300D}

Sie können einen Bericht oder eine Vorlage beim Erstellen eines Projekts oder aus einem vorhandenen Projekt öffnen. Verwenden Sie eine Vorlage, um einen Bericht von Grund auf zu konfigurieren.

<!-- 

t_reports_opening.xml

 -->

Es gibt verschiedene Möglichkeiten, einen Bericht zu öffnen:

* Suchen Sie auf der [!UICONTROL New Report] Seite nach einem Bericht oder wählen Sie eine Vorlage.
* Klicken Sie **[!UICONTROL Reports]** im Menü auf und wählen Sie dann einen Bericht oder eine Berichtsvorlage aus.
* Launch a report from a dimension: right-click a dimension name, then select **[!UICONTROL Run Report]** > **`report name`**.
